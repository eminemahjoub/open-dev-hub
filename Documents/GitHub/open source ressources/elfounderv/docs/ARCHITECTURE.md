# Project Architecture & Conventions

This file explains basic conventions used across the repository to keep content consistent and easy to navigate.

## Folder conventions

- `ai-ml/` — AI & machine learning guides and project starter kits.
- `web-development/` — Frontend/backend tutorials and templates.
- `cybersecurity-ctf/` — Challenges, write-ups, and tooling notes.
- `dev-tools-automation/` — Scripts, GH Action templates and automation examples.
- `iot-embedded/` — Hardware projects, wiring diagrams, and firmware examples.

## Contribution conventions

- Add content as Markdown files in the appropriate folder.
- Use relative links for local resources.
- Include attribution and licenses for included third-party content.

## File naming

- Prefer `kebab-case` for folder and file names, e.g., `fastapi-starter.md`.
- Use `README.md` in each folder for overview and curated lists.

## Licensing

- Default license: MIT. For contributed code, prefer permissive licenses unless otherwise stated.
