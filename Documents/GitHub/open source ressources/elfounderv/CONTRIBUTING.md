# Contributing to elfounderv

Thanks for your interest in contributing — whether it's a single helpful link, a tutorial, or a full project template. We welcome contributions of all sizes.

## How to contribute

1. Fork this repository.
2. Create a branch named `category/brief-description` (for example `ai-ml/add-dataset-link`).
3. Add or update content in the appropriate folder (`ai-ml`, `web-development`, `cybersecurity-ctf`, `dev-tools-automation`, `iot-embedded`).
4. Add tests or examples if relevant.
5. Open a Pull Request with a clear title and description of what you changed and why.

### Content guidelines

- Prefer high-quality, authoritative resources (official docs, well-maintained OSS, recognized blogs/tutorials).
- Provide a short annotation for each link (1–2 sentences explaining why it's useful).
- Avoid spam and link-only PRs: add value via summaries or examples where possible.

### Adding a project

If you're adding a full project or template, include:
- A short description and target audience.
- Installation and quick start steps.
- License and attribution for any third-party assets.

### Review process

- PRs will be reviewed for accuracy, usefulness, and formatting.
- Maintainers may suggest edits or request additional details.

### Code style

- Use Markdown for documentation. Keep formatting consistent.
- If you add code samples, include language-specific fenced code blocks and minimal reproducible examples.

### Community

Be respectful, patient, and inclusive. See `CODE_OF_CONDUCT.md` for details.
